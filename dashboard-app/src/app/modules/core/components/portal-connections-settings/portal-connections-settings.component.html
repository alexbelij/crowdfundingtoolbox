PORTAL CONNECTIONS
PORTAL URL

It is URL of your portal. We recommend to use SSL certificate on both sides (Toolbox app and your portal). Take a note, this URL will be checked during the toolbox request from portal for CORS. Only this portal (with included URL) will have access to data.

TOOLBOX SCRIPT

It is necessary part of implementation. By this script, all widgets and user management sites are displayed on portal. Please, add this script before closing BODY tag:

<script src="{{backendUrl}}/public/js/app.js"> </script>
TOOLBOX STYLE

It is necessary part of implementation. By this style file, all user management sites are styled on portal. Please, add this script before closing HEAD tag:

<link rel="stylesheet" href="{{backendUrl}}/public/css/app.css" />
PLACEHOLDERS

Here are HTML placeholders which are necessary part of implementation. You need to copy these placeholders to your portal templates on place, where you wsant to show widgets.

Landing page widget

Main monetization widget situated on full screen.
<div id="cr0wdfundingToolbox-landing'"></div>

Sidebar widget

Classic sidebar widget, defualt set on 300x600.
<div id="cr0wdfundingToolbox-sidebar'"></div>

Leaderboard widget

Full width banner depend on parent container.
<div id="cr0wdfundingToolbox-leaderboard'"></div>

Fixed widget

Fixed/Cookie widget which has fixed position on full screen and is situated on top or on bottom of page.
<div id="cr0wdfundingToolbox-fixed'"></div>

Popup widget

Classic popup banner.
<div id="cr0wdfundingToolbox-popup'"></div>

Locked article widget

Locked article widget is situated in articles and it hide second half of whole content of articles.
<div id="cr0wdfundingToolbox-locked'"></div>

Article widget

Simple article widget. It can be only text with hyperlinks, or simple banner without CTA button.
<div id="cr0wdfundingToolbox-article'"></div>

Custom HTML widget

HTML widget with inline CSS (inline styles are necessary for this).
<div id="cr0wdfundingToolbox-custom'"></div>
